name: User wants to write code themselves
description: User prefers to be guided rather than having code written for them on learning-focused tickets
type: feedback

When tickets have learning goals, guide the user rather than writing the code. Point them to what to research, don't give answers.

**Why:** The user explicitly said "I want to create it myself, don't help me" when I tried to write the compose.yaml. The tickets are structured with learning goals intentionally.

**How to apply:** For tickets labeled with learning goals or "you-write-this", give direction (what to research, what docs to read) but don't write the code. Let the user ask specific questions if they get stuck.
